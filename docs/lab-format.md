# Формат лабораторной работы

Этот формат используется для всех ЛР в репозитории. Его задача — сделать занятие
повторяемым: студент понимает теорию, запускает стенд, получает наблюдаемый результат
и фиксирует выводы.

## Структура README лабораторной

1. **Цель** — один практический навык, который должен появиться после ЛР.
2. **Теория** — короткая модель: термины, причинно-следственная связь, что смотреть в выводе.
3. **Предусловия** — состояние стенда и команды проверки перед стартом.
4. **Задания** — небольшие шаги от наблюдения к изменению конфигурации.
5. **Expected output** — пример нормального вывода, не полный лог.
6. **Критерий успеха** — проверяемые результаты и вопросы на понимание.

## Учебный цикл

Используйте один и тот же цикл для каждой ЛР:

```text
Прочитать теорию -> предсказать вывод -> выполнить команду -> сравнить с expected output
-> объяснить расхождение -> исправить или зафиксировать вывод.
```

Такой порядок важен: в сетевых лабораториях команда сама по себе ничего не доказывает.
Доказательство — это связь между control plane, data plane и наблюдаемым трафиком.

## Минимальный формат отчета

```markdown
# Отчет по ЛР<N>

## 1. Цель
Коротко: что проверялось.

## 2. Схема
Топология, адреса, роли узлов.

## 3. Ход работы
Команда -> важный фрагмент вывода -> вывод.

## 4. Проверка
Ping/traceroute/show/tcpdump/pcap, которые подтверждают результат.

## 5. Ответы на контрольные вопросы
Только по сути: причина, механизм, где это видно.

## 6. Ошибки и исправления
Симптом -> root cause -> fix.
```

## Правила хорошей лабораторной

- Один шаг меняет одну вещь: так проще найти причину ошибки.
- В каждом блоке должен быть наблюдаемый артефакт: маршрут, соседство, SID, pcap, лог.
- Любой `show` нужно связывать с уровнем системы: FRR RIB, kernel FIB, packet capture.
- Раздел `Expected output` показывает ориентир, но студент должен объяснить свой реальный вывод.
- В конце ЛР всегда должна быть очистка или понятное состояние для следующей ЛР.

## Быстрые проверки перед занятием

```bash
make deploy
make verify
```

Для SRv6-блока:

```bash
make srv6
docker exec clab-srv6-r1 vtysh -c "show segment-routing srv6 locator"
```

Для VPN-блока:

```bash
make vpn
docker exec clab-srv6-r1 vtysh -c "show bgp summary"
```
